# mbo-analyst スコアリング基準 v3

## 5 軸評価

MBO 候補銘柄を以下の 5 軸で評価する。

| 軸 | 名称 | スコア方向 |
|---|---|---|
| A | 買収コスト（Acquisition Cost） | 高いほど割安 |
| B | 事業品質（Business Quality） | 高いほど高品質 |
| C | 支配構造（Control Structure） | 高いほど MBO 実現容易 |
| D | 実行容易性（Deal Feasibility） | 高いほどディール成立容易 |
| E | 阻害要因（Impediment Score） | 高いほど障害が大きい（逆方向） |

```
入力: CSV の「業種」列（東証業種名）
```

## A: 買収コスト（Acquisition Cost）

### A-1: PBR 閾値（業種別）

| スコア | α IT型 | β SVC型 | γ 加工型 | δ 素材型 | ε 資産型 |
|--------|--------|---------|---------|---------|---------|
| 5 | ≤1.0 | ≤0.6 | ≤0.5 | ≤0.4 | ≤0.5 |
| 4 | ≤1.5 | ≤1.0 | ≤0.8 | ≤0.6 | ≤0.7 |
| 3 | ≤2.5 | ≤1.5 | ≤1.0 | ≤0.8 | ≤1.0 |
| 2 | ≤4.0 | ≤2.5 | ≤1.5 | ≤1.2 | ≤1.5 |
| 1 | >4.0 | >2.5 | >1.5 | >1.2 | >1.5 |

※ 閾値は暫定。データ検証で修正する。

### A-2: EV/EBITDA 閾値（業種別）

| スコア | α IT型 | β SVC型 | γ 加工型 | δ 素材型 | ε 資産型 |
|--------|--------|---------|---------|---------|---------|
| 5 | ≤8 | ≤5 | ≤5 | ≤4 | ≤5 |
| 4 | ≤12 | ≤8 | ≤8 | ≤6 | ≤8 |
| 3 | ≤16 | ≤10 | ≤10 | ≤8 | ≤10 |
| 2 | ≤20 | ≤12 | ≤12 | ≤10 | ≤12 |
| 1 | >20 | >12 | >12 | >10 | >12 |

※ 閾値は暫定。データ検証で修正する。

### A-3: FCF 利回り閾値（全プロファイル共通）

| スコア | 閾値 |
|--------|------|
| 5 | >10% |
| 4 | >6% |
| 3 | >4% |
| 2 | >2% |
| 1 | ≤2% |

FCF 利回り = FCF / 時価総額 × 100（単位を揃えること）

### A-4: FCF 安定性フラグ

```
FCF利回り ÷ 営業CFマージン(営業CF/売上) の比率で判定:
- 比率 0.5-1.5 → "stable"
- 比率 0.5未満 or 1.5超 → "volatile"（設備投資の谷/山の可能性）
- データなし → "unknown"
```

batch-score ではフラグを付記するのみ。スコア算出には影響しない。

### A-5: MBO プレミアム耐性

```
premium_tolerance = (1 / PBR - 1) × 100

例:
- PBR 0.5 → +100%（解散価値の2倍までプレミアム可）
- PBR 0.8 → +25%
- PBR 1.0 → 0%（限界）
- PBR 2.0 → -50%（プレミアムなしでも割高）
```

| スコア | premium_tolerance |
|--------|-------------------|
| 5 | >50% |
| 4 | >20% |
| 3 | >0% |
| 2 | >-30% |
| 1 | ≤-30% |

### A-6: 自己資本比率閾値（全プロファイル共通）

| スコア | 閾値 |
|--------|------|
| 5 | >60% |
| 4 | >50% |
| 3 | >30% |
| 2 | >20% |
| 1 | ≤20% |

### A スコア算出: 業種別ウエイト配分

| 構成指標 | α IT型 | β SVC型 | γ 加工型 | δ 素材型 | ε 資産型 |
|---------|--------|---------|---------|---------|---------|
| EV/EBITDA | 30% | 20% | 20% | 15% | 15% |
| PBR | 10% | 20% | 20% | 25% | 30% |
| FCF 利回り | 25% | 25% | 25% | 25% | 25% |
| プレミアム耐性 | 10% | 15% | 15% | 15% | 15% |
| 自己資本比率 | 25% | 20% | 20% | 20% | 15% |

```
A = Σ(指標スコア × ウエイト)
小数点 1 位まで保持。1.0-5.0 にクランプ。
```

## B: 事業品質（Business Quality）

### B-1: 営業利益率閾値（業種別）

| スコア | α IT型 | β SVC型 | γ 加工型 | δ 素材型 | ε 資産型 |
|--------|--------|---------|---------|---------|---------|
| 5 | >20% | >15% | >15% | >12% | >15% |
| 4 | >15% | >10% | >10% | >8% | >10% |
| 3 | >10% | >7% | >5% | >5% | >7% |
| 2 | >5% | >3% | >0% | >2% | >3% |
| 1 | ≤5% | ≤3% | ≤0% | ≤2% | ≤3% |

※ 閾値は暫定。データ検証で修正する。

### B-2: 調整 ROE

```
調整ROE の算出:
- 自己資本比率 ≤ 60% の場合: 調整ROE = ROE（そのまま）
- 自己資本比率 > 60% の場合: 調整ROE = ROE × (自己資本比率 / 60%)
```

| スコア | 調整 ROE |
|--------|----------|
| 5 | >15% |
| 4 | >10% |
| 3 | >8% |
| 2 | >5% |
| 1 | ≤5% |

### B-3: 資本効率ボーナス

```
自己資本比率 > 70%: B スコアに +0.3 加算
自己資本比率 > 80%: B スコアに +0.5 加算
```

### B スコア算出

```
B = (営業利益率スコア + 調整ROEスコア) / 2 + 資本効率ボーナス
小数点 1 位まで保持。1.0-5.0 にクランプ。
```

## C: 支配構造（Control Structure）

C は 2 つのサブ軸で構成する。

### C1: MBO 能力（Ability）

| スコア | 条件 |
|--------|------|
| 5 | 経営者/創業家が 50% 超保有。スクイーズアウト技術的に可能 |
| 4 | 経営者/創業家が 30-50% 保有。友好的株主含め過半数に到達可能 |
| 3 | 経営者/創業家が 15-30% 保有。PE ファンドとの組成で実行可能 |
| 2 | 経営者/創業家が 5-15% 保有。サラリーマン経営に近い |
| 1 | 経営者保有 5% 未満。または支配株主不在 |

**「経営者/創業家」の範囲:**

```
以下を全て合算して持株比率とする:
1. 経営者本人（代表取締役、会長、社長）
2. 創業家の血縁者（2 親等以内）
3. 上記が支配する資産管理会社・財団
4. 上記が実質支配する法人（名義上の筆頭株主含む）

※ 親会社・持株会社が上場企業の場合は「親会社 TOB 型」として別枠
※ 従業員持株会は「友好株主」であり、上記合算には含めない
```

**補助指標:**
- 友好株主比率: 上記 + 従業員持株会 + 明確に友好的な法人株主の合計
- 浮動株比率: 1 - (友好株主比率 + 自己株式比率)
- 時価総額: 500 億円以下 → 有利、1000 億円超 → 不利

### C2: MBO 動機（Motivation）— 時間軸別に分類

C2 サブ項目を **短期催化（C2_short）** と **中期構造（C2_mid）** に分類して評価する。

#### C2_short: 短期催化（0-6ヶ月以内の MBO トリガーに直結する動機）

| チェック項目 | 配点 | 判定基準 |
|-------------|------|----------|
| 後継者・世代交代 | +2.0 | 創業者が 60 歳超かつ後継者が非上場化を望む可能性。or 後継者問題が公知 |
| 対市場ストレス（PBR 不満） | +1.5 | PBR < 1.0 で経営者が株価に不満を持つ合理性がある |
| 外部圧力（アクティビスト） | +1.5 | アクティビスト保有 5% 超、または株主還元要求の記録がある |

```
C2_short 合計配点 → スコア変換（1.0-5.0）:
0.0       → C2_short = 1
0.5-1.0   → C2_short = 2
1.5-2.0   → C2_short = 3
2.5-3.5   → C2_short = 4
4.0-5.0   → C2_short = 5
```

#### C2_mid: 中期構造（6-24ヶ月の構造変化に関連する動機）

| チェック項目 | 配点 | 判定基準 |
|-------------|------|----------|
| 上場維持効用の低さ | +2.0 | 時価総額 200 億円以下でスタンダード市場 → 上場コストが相対的に重い |
| 非公開化便益 | +1.5 | 業界再編トレンド。同業他社で MBO/TOB が直近 2 年内に発生 |
| 東証要請への対応 | +1.0 | PBR 1 倍割れで東証の資本効率改善要請の対象 |
| 規制・レピュテーション | +0.5 | 業界の風評リスクから非公開化が合理的 |

```
C2_mid 合計配点 → スコア変換（1.0-5.0）:
0.0       → C2_mid = 1
0.5-1.0   → C2_mid = 2
1.5-2.0   → C2_mid = 3
2.5-3.5   → C2_mid = 4
4.0-5.0   → C2_mid = 5
```

#### C2 統合

```
C2 = C2_short × 0.5 + C2_mid × 0.5
小数点 1 位まで保持。
```

#### 動機不足ガードレール

C2 < 2.5 の場合、Tier 上限を B に制約する（過剰評価防止）。
レポートに「動機不足ガードレール適用」を注記する。

### C スコア統合

```
C = C1 × 0.6 + C2 × 0.4
小数点 1 位まで保持。
```

### C 属性

| 属性 | 値 | 説明 |
|------|-----|------|
| confidence | high / medium / low | 株主構成の調査確度 |
| mbo_type | 下記参照 | MBO の類型 |
| trigger | 文字列 | MBO の最も可能性の高い契機 |

#### mbo_type 分類体系

| mbo_type | 定義 |
|----------|------|
| founder_mbo | 創業家が経営に関与し 30% 超保有 |
| family_holding | 創業家が資産管理会社経由で支配 |
| parent_tob | 親会社/持株会社が支配的株主 |
| pe_buyout | PE ファンドが経営陣と組んで実行 |
| activist_trigger | アクティビスト圧力が MBO/TOB のトリガー |
| industry_consolidation | 同業他社による TOB が現実的 |
| unsuitable | 構造的に MBO が成立しない |

## D: 実行容易性（Deal Feasibility）

### D-1: LBO 余力

```
lbo_capacity = (EBITDA × 6) / (時価総額 × 1.4)

lbo_capacity ≥ 1.5 → スコア 5
lbo_capacity ≥ 1.0 → スコア 4
lbo_capacity ≥ 0.7 → スコア 3
lbo_capacity ≥ 0.4 → スコア 2
lbo_capacity < 0.4 → スコア 1
```

### D-2: 時価総額ファクター

| スコア | 時価総額 |
|--------|----------|
| 5 | 100 億円以下 |
| 4 | 300 億円以下 |
| 3 | 500 億円以下 |
| 2 | 1000 億円以下 |
| 1 | 1000 億円超 |

### D-3: 浮動株ファクター

| スコア | 浮動株比率 |
|--------|-----------|
| 5 | 30% 以下 |
| 4 | 40% 以下 |
| 3 | 50% 以下 |
| 2 | 60% 以下 |
| 1 | 60% 超 |

### D-4: 資金調達実行性（financing_feasibility）

#### 想定 LTV

```
LTV = EV / (有形固定資産 + 投資有価証券)

LTV < 0.5  → スコア 5（担保余力が十分）
LTV < 0.7  → スコア 4
LTV < 1.0  → スコア 3
LTV < 1.5  → スコア 2
LTV ≥ 1.5  → スコア 1
```

#### DSCR（元利返済カバレッジ比率）

```
DSCR = EBITDA / (EV × 0.03)   ← 推定借入コスト 3%

DSCR ≥ 3.0  → スコア 5
DSCR ≥ 2.0  → スコア 4
DSCR ≥ 1.5  → スコア 3
DSCR ≥ 1.0  → スコア 2
DSCR < 1.0  → スコア 1（返済困難）
```

#### 担保余力

```
collateral_margin = 有形固定資産 + 投資有価証券 − 既存有利子負債

正値でマージンが大きいほど高スコア（LTV スコアの参考値として活用）
```

#### 金利耐性

```
interest_tolerance = EBITDA / (EV × 金利水準)

≥ 1.0  → 高（高金利局面でも返済可能）
0.5-1.0 → 中（標準的）
< 0.5  → 低（高金利局面でリスク）
```

#### 金利局面補正（マクロ条件）

**基準金利**: 短期プライムレート（短プラ）+ スプレッド中央値 +3.0% = **LBO シニアローン想定金利**

実務上の調達コスト感: 短プラ + 2-4%（案件・格付けにより幅あり）。中央値として +3% を基準値とする。
参照タイミング: TOB 公表日（または analyze 実施日）時点の短プラを使用する。
政策金利・TIBOR は使用しない（実際の MBO 調達コストを反映しないため）。

| LBO シニアローン想定金利 | D スコア補正 |
|---|---|
| < 3.5%（超低金利局面） | +0.2 |
| 3.5-5.5%（標準局面） | ±0.0 |
| ≥ 5.5%（高金利局面） | -0.2 |

目安（参考）:
- 短プラ < 0.5%    → 想定金利 < 3.5% → +0.2（ゼロ金利政策下）
- 短プラ 0.5-2.5% → 想定金利 3.5-5.5% → ±0.0  ← 2025年（短プラ 1.625%）
- 短プラ ≥ 2.5%   → 想定金利 ≥ 5.5% → -0.2

financing_feasibility スコア = LTV スコア × 0.35 + DSCR スコア × 0.40 + 担保余力補正 × 0.25

### D スコア算出

```
D_base = LBO余力スコア × 0.3 + 時価総額スコア × 0.2 + 浮動株スコア × 0.2 + financing_feasibility スコア × 0.3
D = D_base + 金利局面補正
小数点 1 位まで保持。
```

## E: 阻害要因（Impediment Score）

注: **E は加算式。高いほど阻害が大きい。** A-D とは逆方向。

### 致命的要因（各 +3.0）

| 要因 | 説明 |
|------|------|
| MBO 主体不在 | 経営陣持株 5% 未満で創業家も不在 |
| コンソーシアム構造 | 3 社以上の対等な法人株主が合意必要 |
| 法的障壁 | 業法上の株主資格要件、当局許認可が MBO を阻害 |
| 持分法適用の壁 | 大株主が持分法で連結しており、売却意思なし |

### 重大要因（各 +2.0）

| 要因 | 説明 |
|------|------|
| 主要株主の反対リスク | 機関投資家/アクティビストが 10% 超保有し、安値 MBO 反対の可能性 |
| 高バリュエーション | PBR 2.0 倍超で MBO プレミアム負担が過大 |
| 業績悪化中 | 直近で売上 or 利益が 20% 超減少 |
| 巨額の有利子負債 | 自己資本比率 30% 未満で追加レバレッジ困難 |

### 注意要因（各 +1.0）

| 要因 | 説明 |
|------|------|
| 業界構造縮小 | 市場全体が縮小トレンド |
| 高配当 | 配当利回り 5% 超で TOB プレミアムが高額化 |
| アクティビスト存在 | 5% 超保有のアクティビスト/投資ファンド |
| 流動性リスク | スタンダード市場で出来高極小 |
| 労務リスク | 労働集約型で人材流出リスク |
| 外国人株主 | 外国人持株 20% 超で手続き/適正価格の圧力 |

### E スコア算出

```
E = Σ(致命的要因 × 3.0) + Σ(重大要因 × 2.0) + Σ(注意要因 × 1.0)

E の範囲: 0（阻害なし）〜 理論上上限なし
実用上: 0-3 = 小、4-6 = 中、7-9 = 大、10+ = 致命的
```

## MBO 総合スコア（MCS）とティア判定

### MCS 算出

```
MCS = (A + B + D) × C_factor - E_penalty

C_factor:
  C ≥ 4.0 → 1.2
  C ≥ 3.0 → 1.0
  C ≥ 2.0 → 0.7
  C < 2.0 → 0.4

E_penalty:
  E = 0    → 0
  E = 1-3  → 1.0
  E = 4-6  → 2.5
  E = 7-9  → 5.0
  E ≥ 10   → 8.0
```

### ティア判定

| ティア | 条件 | 意味 |
|--------|------|------|
| **Tier S** | C1 ≥ 4 かつ C2 ≥ 3 かつ A ≥ 4.0 かつ E < 4 | MBO が「いつ起きてもおかしくない」 |
| **Tier A** | C1 ≥ 3 かつ (A+B) ≥ 8.0 かつ E < 7 | MBO 有力候補 |
| **Tier B** | C1 ≥ 2 かつ (A+B) ≥ 7.0 かつ E < 10 | MBO の可能性あり。条件次第 |
| **Tier C** | C1 ≤ 1 または E ≥ 10 | MBO 困難 |

**confidence による補正:**
- confidence = high → ティアそのまま
- confidence = medium → ティアを 1 段階下げて「暫定」表示
- confidence = low → Tier B 以上にはなれない

**動機不足ガードレール:**
- C2 < 2.5 → Tier 上限 B（confidence 補正よりも優先して適用）

## スクリーニング 3 層構造

### L1: 業種別ゲート条件

L1 を通過しない銘柄はスコアリング対象外。

| プロファイル | ゲート条件 | 理由 |
|-------------|-----------|------|
| α IT型 | 営業利益率 > 0% かつ FCF > 0 | IT の MBO は安定キャッシュフローが前提 |
| β SVC型 | 自己資本比率 > 20% かつ 営業利益率 > 0% | サービス業 MBO は LBO 前提。最低限の財務基盤が必要 |
| γ 加工型 | 自己資本比率 > 20% | 標準的な財務健全性 |
| δ 素材型 | PBR < 1.5 かつ 自己資本比率 > 20% | 素材型で PBR 1.5 超は MBO プレミアム負担が非現実的 |
| ε 資産型 | PBR < 2.0 かつ 自己資本比率 > 15% | 資産型は NAV がベース |

全プロファイル共通:
- 時価総額 ≤ 1000 億円
- MBO 対象 = × の業種は除外

### L2: 業種別フィルタ（ゆるいバリュエーション条件）

| 条件 | α IT型 | β SVC型 | γ 加工型 | δ 素材型 | ε 資産型 |
|------|--------|---------|---------|---------|---------|
| PBR | ≤4.0 | ≤2.5 | ≤1.5 | ≤1.2 | ≤1.5 |
| EV/EBITDA | ≤20 | ≤12 | ≤12 | ≤10 | ≤12 |
| 自己資本比率 | ≥20% | ≥20% | ≥20% | ≥20% | ≥15% |
| 時価総額 | ≤1000億円 | ≤1000億円 | ≤1000億円 | ≤1000億円 | ≤1000億円 |

### L3: 業種別重視指標

A スコアの業種別ウエイト配分をスクリーニング段階から適用する。

| プロファイル | 最重視指標 | 理由 |
|-------------|-----------|------|
| α IT型 | EV/EBITDA, 営業利益率 | PBR は構造的に高く判別力なし |
| β SVC型 | FCF 利回り, 営業利益率 | サービス業は FCF 安定性が鍵 |
| γ 加工型 | EV/EBITDA, PBR | バランス型 |
| δ 素材型 | PBR, 自己資本比率 | PBR が NAV を反映。利益率は構造的に低い |
| ε 資産型 | PBR, FCF 利回り | 不動産は NAV ベース |

### スクリーニング処理フロー

```
CSV 入力
  ↓
[東証業種を読み取り → プロファイル割当]
  ↓
[L1: 業種別ゲート条件] ← 不適格銘柄を除外
  ↓
[L2: 業種別フィルタ]   ← ゆるいバリュエーション条件で候補を絞る
  ↓
[L3: 業種別ウエイトで A/B スコア算出]
  ↓
スコア順にソートして出力
```

## CSV ベーススコアリング（batch-score 用）

batch-score 操作および csv-data モードの analyze で使用する。
CSV の財務指標列 + 業種列から A スコア（valuation_score）を算出する。

### 算出アルゴリズム

#### Step 0: 業種プロファイル判定

CSV の「業種」列から東証業種を読み取り、マッピングテーブルでプロファイルを割り当てる。業種列がない場合は γ（加工型）をデフォルト適用し、警告を出力する。

#### Step 1: L1 ゲート判定

業種別ゲート条件に不合格 → A = 0（対象外）として結果に含めるが、ランキングからは除外。

#### Step 2: 個別指標スコア（業種別閾値で 1-5）

A-1 〜 A-6 の各閾値テーブルを参照し、当該プロファイルの閾値でスコアリングする。

- 指標が CSV にない場合 → その指標を除外して残りで計算
- 数値が欠損（空欄、"-"、"N/A" 等）の場合 → その指標を除外

#### Step 3: A スコア算出

業種別ウエイト配分（A スコア算出テーブル参照）で加重平均を算出する。

```
A = Σ(指標スコア × 業種別ウエイト)
利用可能な指標のウエイトで再正規化する（欠損指標のウエイトを残りに按分）。
小数点 1 位まで保持。1.0-5.0 にクランプ。
```

#### Step 4: B スコア算出（可能な場合）

CSV に営業利益率・ROE・自己資本比率がある場合、B スコアも算出する。

```
B = (営業利益率スコア + 調整ROEスコア) / 2 + 資本効率ボーナス
```

B スコアが算出不能の場合は null とする。

### 注意事項

- CSV ベーススコアリングは Web 検索で得られる定性情報（業績トレンド、特殊要因等）を反映しない
- 結果は一次スクリーニング用であり、最終的な判断には analyze（5 軸評価）が必要
- FCF 利回りの計算で時価総額の単位に注意する（百万円 or 億円）
- 業種列がない場合のデフォルト（γ 加工型）は中立的な閾値であり、過度に楽観/悲観にならない

## P_Score: TOB プレミアム潜在力

Gate 通過銘柄に対してのみ算出する。
A/D 評価で収集済みの財務データから計算し、追加の Web 検索は不要。

### 構成指標と重み

| 指標 | 算出式 | 重み（通常） | 重み（hidden_asset 不明時） |
|------|--------|------------|---------------------------|
| nav_discount | max(0, 1/PBR − 1) | 40% | 47% |
| net_cash_ratio | max(0, ネット現金 / 時価総額) | 30% | 35% |
| hidden_asset_coeff | 土地等含み益 / 時価総額（判明分） | 15% | 除外 |
| fcf_yield | max(0, FCF / 時価総額) | 15% | 18% |

ネット現金 = 現金及び現金同等物 + 短期有価証券 − 有利子負債合計

hidden_asset_coeff が不明（データなし）の場合、その指標を除外し残り 3 指標のウエイトで再正規化する。

### nav_discount スコア（1-5）

| スコア | nav_discount 値（1/PBR − 1） | 対応 PBR |
|--------|------------------------------|---------|
| 5 | ≥ 1.0 | ≤ 0.50 |
| 4 | ≥ 0.5 | ≤ 0.67 |
| 3 | ≥ 0.25 | ≤ 0.80 |
| 2 | ≥ 0.05 | ≤ 0.95 |
| 1 | < 0.05 | > 0.95 |

### net_cash_ratio スコア（1-5）

| スコア | 閾値 | 備考 |
|--------|------|------|
| 5 | ≥ 0.30 | ネット現金 = 時価総額の 30% 以上 |
| 4 | ≥ 0.10 | |
| 3 | ≥ 0.05 | |
| 2 | ≥ 0 | ネット現金プラス |
| 1 | < 0 | ネット有利子負債 |

### hidden_asset_coeff スコア（1-5）

| スコア | 閾値 |
|--------|------|
| 5 | ≥ 0.50 |
| 4 | ≥ 0.30 |
| 3 | ≥ 0.10 |
| 2 | ≥ 0.05 |
| 1 | < 0.05 |

### fcf_yield スコア（1-5）

A-3（FCF 利回り閾値）と同一基準を使用する。

| スコア | 閾値 |
|--------|------|
| 5 | > 10% |
| 4 | > 6% |
| 3 | > 4% |
| 2 | > 2% |
| 1 | ≤ 2% |

### P_Score 算出式

```
# 通常（hidden_asset_coeff が判明している場合）
P_Score = nav_score × 0.40
        + net_cash_score × 0.30
        + hidden_score × 0.15
        + fcf_score × 0.15

# hidden_asset 不明の場合（再正規化）
P_Score = nav_score × 0.47
        + net_cash_score × 0.35
        + fcf_score × 0.18

P_Score = clamp(round(P_Score, 1), 1.0, 5.0)
```

### 受け入れ基準の検証例

PBR < 0.5（nav_score = 5）かつ net_cash > 30%（net_score = 5）の場合、hidden_asset 不明・fcf_yield 最低（score = 1）でも:
```
P_Score = 5×0.47 + 5×0.35 + 1×0.18 = 2.35 + 1.75 + 0.18 = 4.28 ≥ 4 ✓
```

### P_Score の解釈

| P_Score | 解釈 |
|---------|------|
| 4.0-5.0 | 高プレミアム案件。TOB が成立すれば大幅な価格上昇が期待できる |
| 3.0-3.9 | 中程度のプレミアム潜在力 |
| 2.0-2.9 | 低プレミアム。業績改善期待が先行する銘柄 |
| 1.0-1.9 | プレミアム誘引力が乏しい |

## Priority 判定マトリクス

Priority は **短期（Priority_short）** と **中期（Priority_mid）** の 2 系統を独立して算出する。

### MBO_Score 正規化

MCS を 1-5 スケールに変換した値。

```
MBO_Score = clamp(round(MCS / 3.0, 1), 1.0, 5.0)

目安:
  MCS ≥ 15 → MBO_Score = 5.0
  MCS ≥ 12 → MBO_Score = 4.0
  MCS = 9  → MBO_Score = 3.0
  MCS = 6  → MBO_Score = 2.0
  MCS < 6  → MBO_Score = 1.0〜1.9
```

### Priority_short（短期イベントドリブン重視）

軸: C2_short × MBO_Score のマトリクス。閾値: **3.0**。

|  | **MBO_Score ≥ 3.0** | **MBO_Score < 3.0** |
|---|---|---|
| **C2_short ≥ 3.0** | **最優先** | 通常監視 |
| **C2_short < 3.0** | 要確認 | 対象外 |

意味:
- 最優先: 短期催化が明確で MBO 蓋然性も高い → IR・適時開示を週次でモニタリング
- 通常監視: 短期催化はあるが蓋然性は中程度 → 月次レビュー対象
- 要確認: 短期催化は不明確だが MBO_Score は高い → 中期シナリオを確認
- 対象外: 短期での MBO 成立可能性が低い

### Priority_mid（中期構造重視）

軸: C2_mid × P_Score のマトリクス。閾値: **3.5**。

|  | **P_Score ≥ 3.5** | **P_Score < 3.5** |
|---|---|---|
| **C2_mid ≥ 3.0** | **最優先** | 通常監視 |
| **C2_mid < 3.0** | 要確認 | 対象外 |

意味:
- 最優先: 中期構造的な MBO 動機が強くプレミアム潜在力も高い → 四半期レビュー必須
- 通常監視: 中期構造は良好だがプレミアム潜在力が控えめ → 半期レビュー対象
- 要確認: 構造動機は不明確だがプレミアム潜在力が高い（誤検知注意）
- 対象外: 中期での MBO 成立可能性が低い

### authority_signal による引き上げ

authority_signal が検出された場合: 両 Priority を 1 段階引き上げる。

| 引き上げ前 | 引き上げ後 |
|---|---|
| 対象外 | 要確認 |
| 要確認 | 通常監視 |
| 通常監視 | 最優先 |
| 最優先 | 最優先（変化なし） |
