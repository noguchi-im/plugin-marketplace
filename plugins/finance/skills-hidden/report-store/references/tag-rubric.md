## タグ体系の概要

タグはカテゴリ（分類の軸）ごとに定義される。

| カテゴリ | 目的 | 複数選択 | 必須 |
|---|---|---|---|
| content_type | レポート内容の性質を示す | 可 | 不要 |
| aspect | 分析の切り口を示す | 可 | 不要 |

1 つのレポートに複数カテゴリのタグを付けてよい。
判断に迷う場合や曖昧な場合はタグを付けなくてよい。

## content_type タグの判定基準

レポート内容の性質を示すキーワード。1 つのレポートに複数付与できる。

### factual

レポートの主要部分が**観測可能な事実やデータの整理**で構成されている場合。

- 財務数値の列挙・比較
- 公開情報の構造化
- 定量的な計算結果

判定の目安: 「このセクションの主張は、ソースを確認すれば第三者が同じ結論に達するか？」→ Yes なら factual。

### interpretive

レポートに**事実からの推論・解釈・仮説**が含まれている場合。

- データのトレンドからの推論
- 因果関係の推定
- 「〜と考えられる」「〜の可能性がある」といった条件付きの主張

判定の目安: 「この主張には分析者の解釈が入っているか？」→ Yes なら interpretive。

### speculative

レポートに**確度の低い推測や将来予測**が含まれている場合。

- 限定的なデータからの推測
- 複数シナリオの仮定
- 「〜かもしれない」「〜であれば〜だろう」

判定の目安: 「この主張を支えるデータは十分か？」→ No なら speculative。

### opinion-based

レポートに**意見・評価・推奨に近い内容**が含まれている場合。

- 投資判断に直結する見解
- 「〜すべき」「〜が望ましい」
- 価値判断を伴う比較

判定の目安: 「この主張は分析者の価値観に依存するか？」→ Yes なら opinion-based。

### 複合的な場合

多くのレポートは複数の性質を持つ。その場合は該当する content_type を複数付与する。

例: 財務データの整理（factual）+ トレンド分析（interpretive）→ `[factual, interpretive]`

## aspect タグの付与基準

レポートが扱う分析の切り口を示す。初期語彙は定義せず、運用で追加する。

### 追加の判断基準

- 同じ切り口のレポートが 3 件以上になった時点で、タグとして定義を検討する
- タグ名は英小文字・ハイフン区切り（例: `financial-health`, `competitive-position`）

### 候補（参考、未登録）

- valuation, growth, financial-health, competitive-position
- risk, dividend, market-trend, regulatory

## タグ語彙の管理

### 新規タグの追加

1. tags テーブルに INSERT する
2. この tag-rubric.md に判定基準を追記する
3. changelog に記録する

### タグの廃止

1. 該当タグを使用しているレポートがないことを確認する
2. tags テーブルから削除する（または description に「廃止」と記載して残す）
3. この tag-rubric.md から判定基準を削除する
4. changelog に記録する

### 判定基準の変更

1. この tag-rubric.md を更新する
2. 既存レポートのタグを遡及的に変更するかは命令者が判断する
