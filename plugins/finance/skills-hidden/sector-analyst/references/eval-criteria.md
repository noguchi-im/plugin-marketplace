# sector-analyst 評価基準

本ファイルは `sector-analyst` スキルの動作を検証するためのチェックリストです。
SPEC の受け入れ基準を、ユーザが観察可能な入出力の振る舞いとして記述しています。

---

## analyze 正常系

- [ ] Given セクター名の分析依頼（「日本の物流業界を分析して」）が渡された時
      When analyze を実行する
      Then セクターを特定・定義し、業界データを収集し、TAM 分析・業界構造・競争環境・ランキングを含むレポートを生成し、保存 ID（store_id）が発行される

- [ ] Given depth: 概要 が指定された時
      When analyze を実行する
      Then 市場規模概算と上位 5 社の簡潔な比較のみ収集（最大 5 回検索）し、TAM 詳細分析・業界構造分析・ランキングは行わない

- [ ] Given depth: 詳細 が指定された時
      When analyze を実行する
      Then 拡張データ収集（最大 30 回検索）とサブセグメント分析・地域別比較を含む詳細レポートを生成する

- [ ] Given focus: 競争環境 が指定された時
      When analyze を実行する
      Then 競争環境分析（競合テーブル・シェア動向・競争の性質）を重点的に深掘りしたレポートを生成する

## スコープ外

- [ ] Given セクター・業種でない分析依頼（「トヨタを分析して」）が渡された時
      When analyze を実行する
      Then スコープ外であることを報告して終了する

## セクター特定・定義

- [ ] Given 曖昧なセクター指定（「テクノロジー」）が渡された時
      When セクターを特定する
      Then セクターの範囲を明確化し、分析対象とするサブセグメントを定義した上で分析を進める

## TAM 分析

- [ ] Given depth: 標準 以上で市場規模データが収集できた時
      When TAM 分析を実行する
      Then 市場規模・5年 CAGR・セグメント分解・市場集中度が分析される

- [ ] Given 市場規模データが不足している時
      When TAM 分析を実行する
      Then 利用可能なデータから推定し、推定の前提と制約をレポートに明記する

## 競合テーブル

- [ ] Given 競争環境データが収集できた時
      When 競争環境分析を実行する
      Then 主要企業の売上・成長率・マージン・シェア・差別化・バリュエーションを含む競合テーブルが生成される

- [ ] Given 一部の企業データが不足している時
      When 競合テーブルを生成する
      Then 取得できたデータのみでテーブルを構成し、欠損項目を明記する

## セクター内ランキング

- [ ] Given depth: 標準 以上で企業データが収集できた時
      When セクター内ランキングを生成する
      Then 成長性・収益性の観点で企業がランキングされる

- [ ] Given バリュエーション・財務データが不足している時
      When セクター内ランキングを生成する
      Then 取得できた指標のみでランキングし、制約を明記する

## スコアリングとレーティング

- [ ] Given 分析レポートが生成された時
      When スコアリングを実行する
      Then 評価基準に従い market_attractiveness / competitive_intensity / growth_outlook の 3 軸評価が付与され、各軸に根拠が付記される

- [ ] Given 3 軸スコアが付与された時
      When セクターレーティングを決定する
      Then 3 軸を総合し highly attractive / attractive / neutral / cautious / unattractive のいずれかが付与され、根拠が 1-2 文で記述される

- [ ] Given 分析レポートが保存された時
      When 保存が完了する
      Then 分析品質スコアが付与される

## レポート保存

- [ ] Given 分析レポートが生成された時
      When 保存を実行する
      Then レポートがセクタードメイン（sector-analyst）のメタデータで保存される

- [ ] Given 収集結果の追跡 ID がある時
      When 保存を実行する
      Then 収集結果の ID が「使用した入力」として関連付けて記録される

- [ ] Given 保存サービスが利用不可の時
      When analyze を実行する
      Then 保存をスキップし、分析レポートを直接返却する（store_id: null）

## データ収集フィードバック

- [ ] Given 市場データの鮮度に問題がある時
      When フィードバックを評価する
      Then ソース品質（source-quality）タイプのフィードバックが記録される

- [ ] Given セクター分析に必要だが収集されなかったデータカテゴリがある時
      When フィードバックを評価する
      Then パターン不足（pattern-gap）タイプのフィードバックが記録される

- [ ] Given 収集品質に問題がない時
      When フィードバックを評価する
      Then フィードバックは記録されない

## 収集範囲制御

- [ ] Given depth: 標準 で分析依頼された時
      When 収集を実行する
      Then 標準モードの収集範囲（全 7 トピック、最大 15 回検索）でデータが収集される

- [ ] Given depth: 概要 で分析依頼された時
      When 収集を実行する
      Then 概要モードの収集範囲（市場規模・競争環境の 2 トピック、最大 5 回検索）でデータが収集される

- [ ] Given depth: 詳細 で分析依頼された時
      When 収集を実行する
      Then 詳細モードの収集範囲（拡張トピック、最大 30 回検索）でデータが収集される

## 収集結果の不足

- [ ] Given 市場規模データが取得できなかった時
      When 収集結果を確認する
      Then TAM 分析が制限される旨を報告し、続行するか確認する

- [ ] Given データ収集でエラーが発生した時
      When 収集依頼が失敗する
      Then 利用可能な情報のみで分析を続行し、レポートに制約を明記する

## review

- [ ] Given 存在する store_id と concern が渡された時
      When review を実行する
      Then 元レポートを取得し、concern に基づいて再分析し、更新レポートを保存する

- [ ] Given review で追加データが必要な時
      When review を実行する
      Then 追加データを収集し、新データを含めて再分析する

- [ ] Given 存在しない store_id が渡された時
      When review を実行する
      Then エラーを返す

- [ ] Given セクタードメイン以外のレポートの store_id が渡された時
      When review を実行する
      Then スコープ外であることを報告して終了する

## 学習記録

- [ ] Given analyze が完了した時
      When 結果を返却する
      Then 実行ログ（セクター名、分析深度、3軸スコア、セクターレーティング、データ充足状況）が記録される

- [ ] Given review が完了した時
      When 結果を返却する
      Then 実行ログ（セクター名、追跡 ID、変更内容）が記録される
