# finance-advisor 評価基準

本ファイルは `finance-advisor` スキルの動作を検証するためのチェックリストです。
SPEC の受け入れ基準を、ユーザが観察可能な入出力の振る舞いとして記述しています。

---

## 複雑さ判定

- [ ] Given 「最近円安が進んでるけど影響は？」と聞かれた時
      When 複雑さを判定する
      Then simple と判定し、自己回答フローに進む

- [ ] Given 「トヨタの株について詳しく分析して」と聞かれた時
      When 複雑さを判定する
      Then expert と判定し、アナリスト委任フローに進む

- [ ] Given 「半導体セクター全体の動向と主要5銘柄の比較分析をして」と聞かれた時
      When 複雑さを判定する
      Then research と判定し、方針確認フローに進む

- [ ] Given 複雑さの判定に迷った時
      When 判定する
      Then 命令者に確認する

## simple フロー

- [ ] Given 簡単な相談が渡された時
      When 自己回答する
      Then 予算枠内で情報収集・回答し、数値にはソースと日付を明記する

- [ ] Given 既存の関連レポートが存在する時
      When 自己回答する
      Then 既存レポートを参照して回答に活用する

- [ ] Given 一次分析を要する回答が必要な時
      When 自己回答を試みる
      Then 一次分析は行わず、expert フローへの切り替えを提案する

## expert フロー

- [ ] Given 専門分析依頼を受けた時
      When アナリストを選定する
      Then 登録済みアナリスト一覧の capability を照合し、適切なアナリストを選ぶ

- [ ] Given アナリスト一覧から自明に選定できない時
      When アナリストを選定する
      Then アナリスト一覧を最新化し、最新情報で判断する

- [ ] Given 該当するアナリストがいない時
      When アナリストを選定する
      Then 報告し、対応可能な範囲を提示する

- [ ] Given 高コストな操作が必要な expert 依頼を受けた時
      When 予算を見積もる
      Then 予算の内訳を提示し命令者の承認を得てから実行する

- [ ] Given アナリストに依頼する時
      When 分析依頼を構成する
      Then 予算枠を指定してアナリストに渡す

- [ ] Given アナリストの分析が完了した時
      When 結果を返す
      Then 利用記録を残し、レポートを追跡 ID（store_id）付きで提示する

## research フロー

- [ ] Given 大規模調査依頼を受けた時
      When 方針を確認する
      Then 分析対象・深さを命令者に確認する

- [ ] Given 方針が確認された時
      When 予算を提示する
      Then 予算枠に基づき、advisor 分・各アナリスト分・合計の内訳を提示し命令者の承認を得る

- [ ] Given 予算が承認された時
      When 分析計画を立てる
      Then 必要なアナリストと依頼内容・予算配分の計画を命令者に提示し承認を得る

- [ ] Given 複数アナリストの分析が完了した時
      When 結果を統合する
      Then 各出力を比較・統合し、一次分析は行わず統合結果を返す

- [ ] Given 各アナリストの分析が完了した時
      When 利用記録を残す
      Then 各アナリストについて利用記録が記録される

## アナリスト一覧キャッシュ

- [ ] Given 初回起動時にアナリスト一覧が空の時
      When アナリスト一覧を初期化する
      Then 既存のアナリスト一覧ファイルを確認し、未登録のアナリストのみ登録し、一覧をキャッシュする

- [ ] Given アナリスト一覧ファイルが存在する時
      When アナリスト一覧を初期化する
      Then ファイルの内容をキャッシュし、再登録は行わない（登録は非冪等）

- [ ] Given frontmatter に `role: analyst` があるスキルが存在する時
      When アナリストを判定する
      Then そのスキルを分析スキルとして登録候補にする

- [ ] Given frontmatter に `role: infra` があるスキルが存在する時
      When アナリストを判定する
      Then そのスキルをスキップし、登録候補にしない

- [ ] Given 精度低下が指摘された時
      When キャッシュを更新する
      Then アナリスト一覧を最新化し、キャッシュを更新する

## 予算管理

- [ ] Given simple フローで回答する時
      When 予算を管理する
      Then simple 予算枠内で完結し、承認は求めない

- [ ] Given expert フローでアナリストに委任する時
      When 予算を管理する
      Then アナリストに予算枠を指定して渡す

- [ ] Given research フローを実行する時
      When 予算を提示する
      Then 予算の内訳（advisor 分・各アナリスト分・合計）を提示し承認を得る

- [ ] Given アナリストが予算超過を報告した時
      When 追加予算が必要になる
      Then 命令者に追加予算の承認を求める

## パッケージ規約の遵守

- [ ] Given 分析を要する依頼を受けた時
      When 処理する
      Then 自ら一次分析は行わず、アナリストに委任する

- [ ] Given 回答に数値を含める時
      When 回答する
      Then ソースと日付を明記する
