# reit-analyst 評価基準

本ファイルは `reit-analyst` スキルの動作を検証するためのチェックリストです。
SPEC の受け入れ基準を、ユーザが観察可能な入出力の振る舞いとして記述しています。

---

## analyze 正常系

- [ ] Given REIT 名の分析依頼（「日本ビルファンドを分析して」）が渡された時
      When analyze を実行する
      Then REIT を特定し、物件タイプを判別し、REIT データを収集し、タイプ固有分析を含むレポートを生成し、保存 ID（store_id）が発行される

- [ ] Given 銘柄コードの分析依頼（「8951 を分析して」）が渡された時
      When analyze を実行する
      Then 銘柄コードから REIT を特定し、分析レポートを生成する

- [ ] Given depth: 標準 で分析依頼された時
      When analyze を実行する
      Then 標準モードのデータ収集（最大 12 回検索）とポートフォリオ分析・バリュエーション分析が実行される

- [ ] Given depth: 概要 が指定された時
      When analyze を実行する
      Then 基本データのみ収集（最大 3 回検索）し、ポートフォリオ詳細分析・バリュエーション詳細分析は行わない

- [ ] Given depth: 詳細 が指定された時
      When analyze を実行する
      Then 拡張データ収集（最大 18 回検索）と公示地価・物件目視確認を含むバリュエーション分析が実行される

- [ ] Given focus: バリュエーション が指定された時
      When analyze を実行する
      Then バリュエーション分析（鑑定評価検証・賃料ギャップ分析）を重点的に深掘りしたレポートを生成する

## 物件タイプ判別

- [ ] Given オフィス特化型 REIT の分析依頼が渡された時
      When 物件タイプを判別する
      Then 物件タイプ: オフィス と判定し、オフィス固有の評価観点で分析する

- [ ] Given 複数タイプに跨がる REIT の分析依頼が渡された時
      When 物件タイプを判別する
      Then 物件タイプ: 総合型 と判定し、各タイプの比率を記録し、タイプ別に評価する

## バリュエーション検証

- [ ] Given 鑑定CAPレートと市場CAPレートに乖離がある時
      When バリュエーション分析を実行する
      Then 乖離の方向（鑑定が甘い/厳しい）と程度を指摘し、独自NAVを推定する

- [ ] Given 現行賃料がマーケット賃料より低い（アンダーレント）時
      When 賃料ギャップ分析を実行する
      Then 賃料上昇余地としてポジティブ要因に計上する

- [ ] Given 現行賃料がマーケット賃料より高い（オーバーレント）時
      When 賃料ギャップ分析を実行する
      Then 持続性リスクとしてネガティブ要因に計上する

## スコープ外

- [ ] Given REIT でない分析依頼（「トヨタを分析して」）が渡された時
      When analyze を実行する
      Then スコープ外であることを報告して終了する

- [ ] Given 私募 REIT の分析依頼が渡された時
      When analyze を実行する
      Then スコープ外であることを報告して終了する

## 収集結果の不足

- [ ] Given 鑑定評価額・物件データが取得できなかった時
      When 収集結果を確認する
      Then バリュエーション分析が制限される旨を報告し、続行するか確認する

- [ ] Given データ収集でエラーが発生した時
      When 収集依頼が失敗する
      Then 利用可能な情報のみで分析を続行し、レポートに制約を明記する

## スコアリングとレーティング

- [ ] Given 分析レポートが生成された時
      When スコアリングを実行する
      Then 評価基準に従い 5 軸評価（portfolio_quality / financial_health / management_cost / valuation / liquidity_accessibility）が付与され、物件タイプに応じた重み調整が適用され、各軸に根拠が付記される

- [ ] Given 5 軸スコアが付与された時
      When 投資レーティングを決定する
      Then 5 軸を総合し strong buy / buy / neutral / sell / strong sell のいずれかが付与され、根拠が 1-2 文で記述される

- [ ] Given 分析レポートが保存された時
      When 保存が完了する
      Then 分析品質スコアが付与される

## compare

- [ ] Given 2 つの REIT の比較依頼が渡された時
      When compare を実行する
      Then 各 REIT を概要モードで分析し、比較マトリクスと推奨を含む比較レポートを生成する

- [ ] Given 同一物件タイプの REIT 比較が渡された時
      When compare を実行する
      Then タイプ固有指標（稼働率・賃料水準等）を含む比較を行う

- [ ] Given 5 銘柄を超える比較依頼が渡された時
      When compare を実行する
      Then 5 銘柄以下への絞り込みを求める

## レポート保存

- [ ] Given 分析レポートが生成された時
      When 保存を実行する
      Then レポートが REIT ドメイン（reit-analyst）のメタデータで保存される

- [ ] Given 保存サービスが利用不可の時
      When analyze を実行する
      Then 保存をスキップし、分析レポートを直接返却する（store_id: null）

## データ収集フィードバック

- [ ] Given REIT データの鮮度に問題がある時
      When フィードバックを評価する
      Then ソース品質（source-quality）タイプのフィードバックが記録される

- [ ] Given REIT 分析に必要だが収集されなかったデータカテゴリがある時
      When フィードバックを評価する
      Then パターン不足（pattern-gap）タイプのフィードバックが記録される

## review

- [ ] Given 存在する store_id と concern が渡された時
      When review を実行する
      Then 元レポートを取得し、concern に基づいて再分析し、更新レポートを保存する

- [ ] Given REIT ドメイン以外のレポートの store_id が渡された時
      When review を実行する
      Then スコープ外であることを報告して終了する

## NOI ブリッジ分析

- [ ] Given depth: 標準 以上で REIT 分析依頼が渡された時
      When NOI ブリッジ分析を実行する
      Then 前期比 NOI 変動が既存物件増収・新規取得寄与・売却影響・コスト変動に分解され、% と金額の両方で示される

- [ ] Given NOI の前期比データが不足している時
      When NOI ブリッジ分析を実行する
      Then 利用可能な項目のみで部分的に分解し、レポートに制約を明記する

## 金利感応度モデル

- [ ] Given 有利子負債の固定/変動比率データが収集できた時
      When 財務分析を実行する
      Then 変動金利部分に対する金利 +50bp / +100bp / +200bp の DPU（分配金）への影響額が算出される

- [ ] Given 借入満期データが収集できた時
      When 財務分析を実行する
      Then 今後 5 年間の借入満期ラダーが年度別に可視化され、集中リスクが評価される

## 3シナリオ NAV

- [ ] Given depth: 標準 以上でバリュエーション分析を実行する時
      When 独自 NAV を推定する
      Then CAPレート・稼働率・賃料成長率の前提を変動させた Bear/Base/Bull の 3 シナリオ NAV レンジが提示され、各シナリオの前提が明示される

- [ ] Given 3シナリオの前提変数データが不足している時
      When 3シナリオ NAV を推定する
      Then 利用可能な変数のみでシナリオを構成し、固定した変数とその理由をレポートに明記する

## 収集範囲制御

- [ ] Given depth: 標準 で分析依頼された時
      When 収集を実行する
      Then 標準モードの収集範囲（T1-T8 の 8 トピック、最大 12 回検索）でデータが収集される

- [ ] Given depth: 概要 で分析依頼された時
      When 収集を実行する
      Then 概要モードの収集範囲（基本情報・財務・流動性の 3 トピック、最大 3 回検索）でデータが収集される

- [ ] Given depth: 詳細 で分析依頼された時
      When 収集を実行する
      Then 詳細モードの収集範囲（エリア別市場データを含む 10 トピック、最大 18 回検索）でデータが収集される
